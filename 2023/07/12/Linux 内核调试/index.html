1. 思路

项目初期参考了一篇论文，想把它的代码搬来自己实现。但是实验室所用的服务器环境各异，很难完全符合论文里的要求。某些地方不匹配总是让人不放心。因此，为了成功编译论文补丁下的内核，所以最好能有一个完全与之对齐的环境。

我选择使用一个 ubuntu:18.04 的 docker。在 docker 中配置环境，包括 gcc 和各种依赖。然后给特定版本的内核打上论文提供的补丁，编译内核，得到 bzImage。

内核运行还需要根文件系统，这依赖 busybox 构建。可以选择在物理机上安装 busybox，也可以选择在 docker 里安装，还可以新建一个 busybox 的 docker 用于构建根文件系统。考虑到是编译出来的内核使用该文件系统，所以还是使用 docker 内安装的 busybox 生成根文件系统。

有了 bzImage 和根文件系统，就可以使用 qemu 启动虚拟机了。可以选择在物理机上启动 qemu，需要将 bzImage 和根文件系统从 docker 里取出，也可以在 docker 里直接安装 qemu 启动内核。本着一站式的原则，我还是选择在 docker 里启动 qemu。
