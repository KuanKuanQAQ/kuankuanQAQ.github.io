tmux 使用心得

到网上一查，tmux 的指令成堆成片。查阅有余，实用不足。想要快速上手高效工具可能只需要几条指令而已。

0. tmux 的拳头特性

tmux 的拳头特性是维持会话（session）。这来自于 tmux 的 client-server 架构。

tmux 会在后台开启一个服务器端，在这个服务器端里启动一个 session，往往这个 session 中运行了一个 shell。当然，shell 进而可以运行各种程序，那时 session 中就运行了那些程序。

你通过 shell 可以连接 tmux 的服务器端，进入 tmux 建立的 session，看到程序的各种输出，并且也可以和这些进程交互。最重要的是，你可以随意断开与 tmux 服务器端的连接，这不会影响 tmux 建立的各种 session。它们不会因为你断开连接而关闭，其中的程序自然也会持续运行。

这些程序可能是一个还需要再跑一周的实验，可能是一组 docker，也可能是你编辑到一半的 vim。当你重新打开 tmux 连接到一个 session 时，它们还都在那里，程序没有中断，docker 没有退出，vim 甚至还保存着剪切板中的内容。

如果你的 tmux 是运行在远程主机上，那么就是更是一个标准的应用场景了。在你与远程主机断开连接后，tmux 的服务器端会继续运行在远程主机上，一切工作环境都保持原样，等待你下次连接。

如果你有个同事，也在相同的远程主机上开发，你们甚至可以连接到同一个 tmux 服务器上，你们所做的所有修改会同步地在双方终端里显现。

1. 会话 session

在介绍 tmux 的快捷键和指令之前，需要说明 tmux 的快捷键前缀。

tmux 为了防止自己的快捷键和其他软件互相干扰，所以在使用所有快捷键前，都需要先按下快捷键前缀，再按下快捷键。tmux 的默认快捷键前缀是 ctrl+b。

emacs 因为没有类似 vim 的模式设计和类似 tmux 的快捷键前缀设计，导致经常发生快捷键冲突。比如 MarkSet 命令默认绑定到 ctrl+space，和系统的输入法开关冲突。我不想修改软件快捷键的默认设置，于是就不再用 emacs 了QAQ。

2. 窗口 window

一个 session 中可以包含多个 window。这些 window 共同组成了一个工作环境。比如你希望一个 window 用来编辑代码，另一个 window 用来编译和执行。

创建 window：ctrl+b c。create。

关闭当前 window：ctrl+b &。

切换到上一个 window：ctrl+b p。

切换到下一个 window：ctrl+b n。

3. 面板 pane

window 类似于桌面，在一个屏幕上不共存。想要一目了然还需要 pane。它会将当前屏幕分为多个面板，每个面板可以单独操作。

左右划分 pane：ctrl+b %。% 很像是一个 / 分开了两个 o。

上下划分 pane：ctrl+b “。这个不好解释。

光标切换到下一个 pane：ctrl+b o。other。

关闭当前光标所在 pane：ctrl+b x。
